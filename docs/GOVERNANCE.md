# Governance

freepalette is currently maintainer-led.

## Process

- Use issues for bugs.
- Use discussions or design issues for larger ideas.
- Use pull requests for code and documentation changes.
- Larger changes require an RFC-style issue before implementation.
- Maintainers may close issues or pull requests that are outside project scope.

## Scope

Maintainers should preserve the project identity:

- free and open source
- local-first
- no telemetry
- no account or cloud dependency for core behavior
- narrow MVP before broad feature work
